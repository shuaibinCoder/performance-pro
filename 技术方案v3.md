# 百人公司绩效管理系统 · 技术方案 v3

## 一、技术栈

| 层 | 技术 | 说明 |
|----|-----|------|
| 运行时 | Node.js 22+ | 内置 `node:sqlite`，无额外依赖 |
| Web框架 | Express 4 | 轻量、成熟 |
| 数据库 | SQLite（node:sqlite） | 文件型，零配置，适合demo |
| 密码加密 | bcryptjs | 纯JS实现，无需编译 |
| 前端框架 | Vue 3（CDN全局构建） | 无构建步骤，字符串模板 |
| UI库 | Bootstrap 5（本地vendor）| 响应式，组件丰富 |
| 图标 | Bootstrap Icons（本地）| 与Bootstrap配套 |
| 图表 | Chart.js（本地vendor）| 轻量，API简单 |

## 二、目录结构

```
performance-mgmt/
├── server.js          # Express服务器 + 全部API路由
├── db.js              # 数据库初始化 + 表结构 + 种子数据
├── package.json
├── performance.db     # SQLite数据库文件（运行时生成）
└── public/
    ├── login.html     # 入口页，自动demo登录
    ├── app.html       # 主SPA（Vue应用挂载点）
    ├── css/
    │   └── style.css  # 全局样式
    ├── js/
    │   ├── api.js     # API客户端封装
    │   └── main.js    # Vue 3主应用 + 所有组件
    └── vendor/        # 本地依赖（Bootstrap, Vue, Chart.js）
```

**关键：** 使用 `app.html` 而非 `index.html`，避免 `express.static` 默认将 `/` 映射到 `index.html`，造成路由冲突。

## 五、前端架构

### Vue 3 组件树

```
App（主应用）
├── 侧边栏（role-based菜单）
├── 顶部栏（周期标识 + 通知 + 用户信息）
└── 页面内容区
    ├── DashboardView（三种角色差异化内容）
    ├── GoalsView（OKR树形列表 + 进度更新）
    ├── ReviewsView（自评/经理评/查看结果）
    ├── FeedbackView（360反馈发送/接收）
    ├── OneOnOneView（面谈列表 + 笔记）
    ├── ProfileView（历史绩效档案 + Chart.js图表）
    ├── ReportsView（数据报表 + Chart.js图表）（Manager/HR）
    ├── CalibrationView（等级校准表格）（Manager/HR）
    ├── EmployeesView（员工管理）（HR）
    ├── CyclesView（周期管理）（HR）
    └── NotificationsView（通知列表）
```

### 路由策略
- 单页应用，用 `view` 变量控制当前页面组件
- 无前端路由库，通过 `v-if` 控制组件显示
- URL 始终是 `/app`，不做 hash 路由

### 状态管理
- 用户信息存 localStorage + Vue data
- 页面级数据在组件 `mounted()` 时异步加载
- 无全局状态管理库

### 关键设计
1. **避免闪烁**：`#page-cover` 全屏遮罩，Vue mounted后淡出
2. **Token管理**：localStorage存储，`_redirecting` 标志防并发401
3. **Demo登录**：`/` → login.html → autoDemo() → 获取HR token → 跳转 `/app`
4. **角色切换**：顶部Demo bar，调用demo-login切换身份，`window.location.reload()`

## 六、关键实现要点

### 防闪烁方案
```html
<!-- app.html: #page-cover默认显示 -->
<div id="page-cover"><!-- spinner --></div>

<!-- main.js: Vue mounted()后淡出 -->
mounted() {
  // 数据加载完成后
  const cover = document.getElementById('page-cover');
  cover.style.opacity = '0';
  setTimeout(() => cover.remove(), 250);
}
```

### Demo自动登录
```javascript
// login.html
async function autoDemo() {
  const token = localStorage.getItem('token');
  if (token) {
    const me = await fetch('/api/auth/me', { headers: { Authorization: 'Bearer '+token } })
                .then(r => r.ok ? r.json() : null);
    if (me) { localStorage.setItem('user', JSON.stringify(me)); location.href='/app'; return; }
    localStorage.clear();
  }
  const data = await fetch('/api/auth/demo-login?role=hr').then(r => r.json());
  localStorage.setItem('token', data.token);
  localStorage.setItem('user', JSON.stringify(data.user));
  location.href = '/app';
}
```

### 关键结果进度汇总到目标
```javascript
// 当KR更新时，自动计算目标整体进度
const avgProgress = krs.reduce((sum, kr) => sum + kr.progress, 0) / krs.length;
db.prepare('UPDATE goals SET progress = ? WHERE id = ?').run(avgProgress, goalId);
```
