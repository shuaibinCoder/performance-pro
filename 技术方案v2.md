## 三、数据库变更

### 3.1 新增表：`one_on_ones`

```sql
CREATE TABLE IF NOT EXISTS one_on_ones (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  manager_id  INTEGER NOT NULL REFERENCES employees(id),
  employee_id INTEGER NOT NULL REFERENCES employees(id),
  cycle_id    INTEGER REFERENCES cycles(id),
  scheduled_at TEXT,                          -- 计划时间 ISO8601
  status      TEXT DEFAULT 'pending',         -- pending | done
  agenda      TEXT,                           -- 议题（JSON 数组存储）
  notes       TEXT,                           -- 会议笔记
  action_items TEXT,                          -- 行动项（JSON 数组）
  created_at  TEXT DEFAULT (datetime('now','localtime'))
);
```

**action_items JSON 结构**（存为 JSON 字符串）：
```json
[
  {
    "id": "uuid",
    "text": "行动项描述",
    "owner": "员工姓名",
    "due_date": "2025-01-31",
    "done": false
  }
]
```

## 四、新增后端接口

### 4.1 Demo 快速登录

```
GET /api/auth/demo-login?role=hr|manager|employee
```

| 参数 | 说明 |
|------|------|
| `role` | `hr` → 返回 admin@company.com；`manager` → zhang.wei@company.com；`employee` → li.ming@company.com |

**响应**：与 `/api/auth/login` 相同格式 `{ token, user }`

**无需认证中间件**，直接查库获取对应用户，创建 session。

---

### 4.2 1-on-1 面谈接口

#### 获取面谈列表
```
GET /api/oneononos?cycle_id=1
```
- 经理：返回自己主持的所有面谈（JOIN 员工信息）
- 员工：返回自己参与的面谈

#### 新建面谈
```
POST /api/oneononos
Body: { employee_id, cycle_id, scheduled_at, agenda }
```
- 仅限 manager/hr 调用

#### 更新面谈（记录笔记/行动项/状态）
```
PUT /api/oneononos/:id
Body: { notes, action_items, status }
```
- 仅限该面谈的 manager_id 对应用户

### 5.2 `login.html` 变更逻辑

```javascript
// 原逻辑：有 token 就跳转 /app
if (localStorage.getItem('token')) window.location.href = '/app';

// 新逻辑：没有 token 也自动以 HR 身份 demo 登录后跳转
async function autoDemo() {
  if (localStorage.getItem('token')) {
    window.location.href = '/app'; return;
  }
  const data = await fetch('/api/auth/demo-login?role=hr').then(r => r.json());
  localStorage.setItem('token', data.token);
  localStorage.setItem('user', JSON.stringify(data.user));
  window.location.href = '/app';
}
autoDemo();
// 保留登录表单，用于手动登录（不再作为默认首屏）
```
