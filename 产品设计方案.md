# 绩效管理系统产品设计方案

> 面向百人规模公司，基于市场竞品分析，打造轻量、实用、可演示的完整绩效管理体验。

---

## 一、市场竞品分析

### 1.1 主流系统概览

| 产品 | 定位 | 核心亮点 | 适用规模 | 主要不足 |
|------|------|---------|---------|---------|
| **Lattice** | 绩效+发展+敬业度 | 持续反馈、1-on-1、个人发展计划 | 100-5000人 | 价格高，配置复杂 |
| **15Five** | 持续绩效管理 | 周度Check-in、OKR、即时表扬 | 50-2000人 | 功能略散，需整合 |
| **Culture Amp** | 敬业度+绩效 | 员工调研驱动、数据科学分析 | 500人以上 | 偏敬业度，绩效弱 |
| **BambooHR** | HR一体化 | 简洁易用、入职/绩效/薪酬 | 20-500人 | 绩效模块功能浅 |
| **Tita** | OKR专注 | OKR对齐树、进度周报 | 50-1000人 | 国际化不足 |
| **飞书绩效** | OKR+KPI融合 | 与飞书生态深度集成、移动优先 | 50-10000人 | 依赖飞书生态 |
| **钉钉绩效** | KPI为主 | 钉钉内嵌、审批流程强 | 50-5000人 | 偏传统，UX老旧 |
| **SAP SuccessFactors** | 企业级全套 | 人才图谱、继任计划 | 1000人以上 | 实施成本极高 |

### 1.2 共同趋势（2024年）

1. **持续化**：从"年度考核"走向"季度/月度/周度"持续反馈，避免"秋后算账"
2. **轻量化**：表单越来越短，操作路径越来越短，移动端优先
3. **对齐可视化**：个人目标→团队目标→公司目标自上而下可视对齐
4. **数据驱动校准**：AI辅助强制分布建议，减少主观偏差
5. **发展导向**：绩效结果与晋升/薪酬建议直接联动

### 1.3 百人公司特有痛点

| 痛点 | 现象 | 需要的解决方案 |
|------|------|-------------|
| 目标对齐难 | 员工不清楚自己目标如何支撑公司战略 | OKR层级对齐视图 |
| 反馈不及时 | 只有年底才知道自己表现如何 | 持续反馈 + 1-on-1 |
| 评估主观性强 | 经理打分缺乏数据支撑 | 多维度量化 + 360佐证 |
| 校准耗时长 | HR组织跨部门校准会议低效 | 可视化分布调整 |
| 员工参与度低 | 员工视绩效为"被考核"而非"成长工具" | 个人成长档案 |

## 三、新增模块详细设计

### 3.1 Demo 演示模式

**目标**：访问 `http://localhost:3000` 直接进入已登录的 HR Dashboard，无需任何操作。

**交互流程**：
```
用户访问首页 → 未登录 → 自动以 HR 管理员身份 demo 登录 → 进入 Dashboard
页面顶部固定提示栏 → 显示"演示模式" + 3个角色切换按钮
点击「切换为研发经理」→ 角色切换，页面重载
```

**设计规范**：
- 顶部提示栏高度 40px，深色背景（`#1E1B4B`），不影响主体布局
- 3个角色按钮：`HR管理员`（默认）、`研发经理`、`普通员工`
- 当前角色高亮显示

### 3.2 1-on-1 面谈记录

**目标**：支持经理与员工定期面谈，结构化记录议题和行动项。

**核心用户流程**：
```
经理进入「1-on-1」→ 新建面谈（选员工、定时间、设议题）
↓
面谈进行中 → 记录笔记、添加行动项（负责人+截止日期）
↓
标记面谈完成 → 员工可查看自己的面谈记录和行动项
```

**页面结构**：
- **经理视角**：面谈列表（按员工分组）+ 新建面谈按钮 + 状态统计
- **员工视角**：我的面谈记录（只读） + 行动项跟踪

**数据字段**：
- 基本信息：经理、员工、计划时间、状态（待进行/已完成）
- 会议记录：议题（Agenda）、会议笔记（Notes）
- 行动项：条目文本、负责人、截止日期、完成状态

### 3.3 个人成长档案

**目标**：员工可查看自己的绩效历史趋势，形成"成长"感知，而非仅看当前评分。

**页面结构**：
```
┌─────────────────────────────────────┐
│  个人信息卡（头像、职位、入职时长）     │
├────────────────┬────────────────────┤
│ 绩效等级历史    │  综合评分趋势折线图  │
│  Q1 B → Q2 B  │  /\/\/\            │
│  Q3 A → Q4 ?  │                    │
├────────────────┴────────────────────┤
│  目标完成率统计（当前周期）            │
├─────────────────────────────────────┤
│  收到的360反馈摘要（优势词/改进词）    │
├─────────────────────────────────────┤
│  我的行动项（来自1-on-1）            │
└─────────────────────────────────────┘
```

**数据来源**：
- 历史 reviews（多个周期）
- goals 完成情况
- feedback 摘要
- one_on_ones 行动项

## 四、交互设计规范

### 4.1 视觉规范（复用现有）

| 元素 | 规范 |
|------|------|
| 主色 | `#4F46E5`（靛蓝） |
| 侧边栏 | `#1E1B4B`（深紫蓝），宽 240px |
| 卡片 | 白色，圆角 `1rem`，浅阴影 |
| 等级色 | S-金 A-绿 B-蓝 C-红 D-灰 |
| 字体 | 系统默认，标题 `fw-bold` |

### 4.2 空状态设计

每个列表页都有空状态提示（icon + 文字 + 引导操作按钮），避免白板体验。

### 4.3 加载状态

所有异步操作显示 Bootstrap Spinner，避免操作无反馈。

### 4.4 响应式（基本适配）

- 侧边栏在 < 768px 时隐藏（通过 Bootstrap 响应式类处理）
- 卡片网格在小屏幕下改为单列

## 五、演示剧本

> 供产品演示使用，展示完整用户旅程

**场景一：HR 全局视角**
1. 打开网站 → 直接进入 HR Dashboard（无需登录）
2. 查看公司完成率：85% 员工完成自评，6% 经理已评分
3. 进入「员工管理」→ 搜索员工，查看 100 人名单
4. 进入「数据报表」→ 看评级分布、部门对比

**场景二：经理视角（研发部）**
1. 顶部切换「研发经理」→ 页面变为经理视角
2. 进入「1-on-1 面谈」→ 查看本月与团队的面谈计划
3. 新建与员工的面谈 → 填写议题 → 添加行动项
4. 进入「绩效校准」→ 调整团队成员等级分布

**场景三：员工视角**
1. 顶部切换「普通员工」→ 进入员工 Dashboard
2. 进入「个人档案」→ 查看自己历史绩效、收到的反馈
3. 进入「目标管理」→ 更新本季度 OKR 进度
4. 进入「360反馈」→ 给同事提交反馈
