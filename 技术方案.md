### 1.3 关键洞察

1. **中小企业痛点**：大企业产品功能冗余、价格高；轻量级产品深度不够
2. **国内特色需求**：强制分布（末位淘汰）、与钉钉/飞书集成、中文本土化
3. **趋势**：从年度考核 → 季度/月度持续反馈；从KPI唯一 → OKR+KPI混合
4. **百人公司痛点**：流程不规范、评分主观、数据分散、没有校准机制

## 二、百人公司场景定义

### 2.1 公司结构

```
总人数：100人
├── 管理层(8人)：CEO、CTO、CMO、CSO、COO、CFO、CHO + 1名助理
├── 研发部(28人)：技术总监1 + 前端8 + 后端12 + 测试5 + 运维2
├── 产品部(8人)：产品总监1 + 产品经理4 + UI设计3
├── 市场部(10人)：市场总监1 + 品牌3 + 内容3 + 渠道3
├── 销售部(22人)：销售总监1 + 大客户8 + 渠道销售8 + 售前5
├── 运营部(10人)：运营总监1 + 用户运营4 + 活动运营3 + 数据运营2
├── 人力资源部(5人)：HR总监1 + HRBP2 + 招聘2
└── 财务部(5人)：财务总监1 + 会计2 + 出纳1 + 审计1
```

### 2.2 绩效管理模式

- **考核周期**：季度评估（主要）+ 年度综合
- **评估模式**：KPI为主 + OKR辅助（管理层和研发用OKR）
- **评分维度**：工作绩效(60%) + 工作态度(20%) + 能力发展(20%)
- **评分等级**：S(超越期望) / A(优秀) / B(达标) / C(待提升) / D(不达标)
- **强制分布**：S≤10%, A≤25%, B≥40%, C≤20%, D≤5%

### 2.3 考核流程

```
周期启动 → 目标制定(员工) → 目标确认(上级) → 过程跟踪
    → 自评提交 → 上级评分 → 360反馈收集 → 绩效校准会议
    → 结果公示 → 面谈 → 归档
```

## 三、功能规划

### 3.1 功能模块

| 模块 | 员工 | 经理 | HR |
|------|------|------|-----|
| 首页仪表盘 | 个人概览 | 团队概览 | 全司概览 |
| 目标管理 | 制定/追踪个人目标 | 审批/查看团队目标 | 全员目标视图 |
| 绩效评估 | 自评填写 | 评审下属 + 自评 | 管理所有评审 |
| 360反馈 | 发起/接收反馈 | 同左 | 管理反馈配置 |
| 绩效校准 | - | 团队校准 | 公司级校准 |
| 数据报表 | 个人报告 | 团队报告 | 全司报告 |
| 员工管理 | - | - | 完整CRUD |
| 周期管理 | - | - | 创建/控制周期 |

### 3.2 核心业务流程

**目标管理流程：**
- 支持 OKR（目标 + 关键结果）和 KPI（指标 + 目标值）两种模式
- 支持目标对齐（子目标关联上级目标）
- 实时进度更新，可视化进度条

**评估流程：**
- 评估维度：工作质量、工作效率、团队协作、创新能力、（管理能力 for managers）
- 自评 → 经理评 → HR校准 → 最终等级

**360反馈：**
- 发起人选择被评估人和评估者
- 评估者匿名填写
- 被评估人可查看汇总（不可查看具体来源）

## 四、技术架构

### 4.1 技术选型

```
后端：Node.js 18+ + Express 4.x
数据库：SQLite（via better-sqlite3，零依赖，单文件）
前端：Vue 3（CDN，免构建）+ Bootstrap 5.3 + Chart.js 4
认证：Token（存储于内存Map，localStorage存储于客户端）
```

**选型理由：**
- SQLite：零配置，单文件数据库，完美适合演示场景
- Vue 3 CDN：无需构建工具，直接浏览器运行
- Bootstrap 5：企业级UI组件，快速构建专业界面
- 整体：`npm install && npm start` 即可运行

## 五、数据库设计

```sql
-- 部门表
departments (id, name, head_id, color)

-- 员工表
employees (id, name, email, password_hash, department_id, manager_id,
           role[employee/manager/hr], position, joined_at, status, avatar_color)

-- 绩效周期
cycles (id, name, type[quarterly/annual], year, quarter,
        start_date, end_date, self_review_deadline, manager_review_deadline,
        status[draft/active/reviewing/calibrating/closed])

-- 目标（支持OKR和KPI）
goals (id, employee_id, cycle_id, type[OKR/KPI], title, description,
       target_value, actual_value, unit, progress, status, parent_goal_id)

-- OKR关键结果
key_results (id, goal_id, description, target, actual, progress, status)

-- 绩效评估
reviews (id, employee_id, reviewer_id, cycle_id,
         type[self/manager/hr], status[draft/submitted/finalized],
         work_quality, work_efficiency, teamwork, innovation, management,
         overall_score, grade[S/A/B/C/D],
         self_summary, reviewer_comment, strengths, improvements,
         submitted_at)

-- 360反馈
feedback (id, from_id, to_id, cycle_id,
          relationship[peer/subordinate/cross_dept],
          strengths, improvements, rating[1-5],
          is_anonymous, submitted_at)

-- 通知
notifications (id, employee_id, type, title, content, is_read, created_at)
```

## 六、API 设计

```
POST   /api/auth/login              登录
GET    /api/auth/me                 获取当前用户

GET    /api/employees               员工列表（支持分页/筛选）
POST   /api/employees               创建员工
PUT    /api/employees/:id           更新员工
DELETE /api/employees/:id           删除员工

GET    /api/cycles                  周期列表
POST   /api/cycles                  创建周期
PUT    /api/cycles/:id/status       更新周期状态

GET    /api/goals                   目标列表（支持employee_id/cycle_id筛选）
POST   /api/goals                   创建目标
PUT    /api/goals/:id               更新目标进度

GET    /api/reviews                 评估列表
POST   /api/reviews                 提交评估
PUT    /api/reviews/:id             更新评估

GET    /api/feedback                反馈列表
POST   /api/feedback                提交反馈

GET    /api/reports/summary         全司概览
GET    /api/reports/distribution    评级分布
GET    /api/reports/departments     部门对比
GET    /api/reports/employees       员工排名

GET    /api/notifications           通知列表
PUT    /api/notifications/read-all  全部已读
```
